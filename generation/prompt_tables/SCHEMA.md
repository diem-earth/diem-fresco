# Prompt Table Schemas — Stage 3 (v0.2)

Three tables, one record per generation unit. **JSON is canonical; CSVs are
derived, read-only review views** regenerated by
`generation/scripts/build_prompt_tables.py`. Never hand-edit the CSVs.

Every record splits into:

- **derived fields** — computed from stage-2 metadata
  (`decomposition/meta/*.json`, `extremities/outputs/*/final_positions.json`)
  and `generation/generation_config.json`. Overwritten on every rebuild.
  Do not edit. **Derived data must remain derived.**
- **`curated` block** — human-authored artistic/editorial data (prompts,
  motifs, direction, status, overrides). **Preserved across rebuilds**
  (merged back by `id`), so editing them is safe. **Human-authored data must
  remain curated.** Persistence is verified by an automated test:
  `python3 generation/scripts/test_curated_persistence.py`.

Each file carries a `provenance` header: `schema_version`, short sha256 of
each source file (including the generation config), the default margin, and
the builder path.

## The prompt model — model-independent first

Prompt authoring is deliberately split so that artistic work is **not blocked
by model selection**:

| Field | Meaning |
|---|---|
| `concept_prompt` | durable artistic and narrative intent for the unit — written in plain language, independent of any generation model or tool. **Author this first**; it survives model changes. |
| `render_prompt` | the operational prompt adapted to a specific generation model/tool (syntax, weights, trigger words). Written only once a model is chosen; derived from the concept, never the other way round. |
| `negative_prompt` | model-facing exclusions (empty = inherit style-pack default once defined) |
| `model_config_ref` | pointer to the model/tool configuration used (sampler, resolution, conditioning setup). No configs exist yet — TBD. |
| `conditioning_assets` | repo-relative paths of conditioning inputs (mask crops, ink crops, reference boards) |
| `generation_notes` | operational log: retries, seeds, acceptance remarks |

**Migration from v0.1:** the old single `prompt_final` field is retired. Any
non-empty legacy `prompt_final` migrates to `render_prompt` on rebuild (it was
defined as "the prompt actually used for generation"); the builder logs every
migration. Unknown legacy keys are dropped with a loud warning, never
silently. Legacy `context.iconography` (extremities) migrates to
`curated.identity_motifs`.

## Generation margins — configurable, not approved

`generation/generation_config.json` holds `default_generation_margin_px`
(currently 64 — an **initial working default, not an approved value**; the
right margin depends on the model, working resolution, conditioning method
and continuity strategy, all open). Per record:

- `curated.generation_margin_override_px` — optional override (null = default);
- `geometry.effective_margin_px` + `geometry.margin_source`
  (`default` | `curated_override`) — the derived value actually used;
- `geometry.gen_window_px_global` / `gen_window_size_px` — **derived** from
  bbox + effective margin, clamped to the canvas. Changing the config default
  or any override requires a rebuild.

## Status lifecycle (`curated.status`)

```
todo → pilot → drafted → approved → generated → composited
                 (seam ornaments start at: blocked_on_design)
```

`drafted` = concept_prompt written · `approved` = concept (and render prompt,
once a model exists) frozen for generation · `generated` = accepted image(s)
exist · `composited` = accepted into the fresco assembly **and** registered in
`registry/artifacts.yaml` with its process-film preview captured.

## 1. `center_superblocks.json` — 79 records

78 `kind: superblock` + 1 `kind: seine_corridor`. All coordinates **global**.

Derived fields (per record): `geometry` (bbox, gen window + margin fields
above, centroid px/lonlat, area, cell count), `assets` (global-coordinate
mask path, cells ref, **`mask_polarity`** — L5 masks are
`region_white_on_black`, extremity masks are the opposite,
`region_black_on_white`; verified 2026-07-10, consumers must normalize),
`context` (palette_world*, arrondissement, quartier,
purity, secondary quartiers ≥5 %, L4 overlays with containment, block_grain
fine/medium/coarse, neighbors with shared-boundary px, seine/canvas-edge
flags, unit flags), `prompt_scaffold` — a **factual unit summary only** (no
artistic direction; it seeds concept authoring and is regenerated).

\* `palette_world` groups units by L1 bank membership. It is a **provisional
artistic grouping** (hypothesis, not an approved palette) — see the plan's
"Provisional artistic hypotheses".

Curated block (canonical field list):

```
status, pilot, style_pack,
concept_prompt, render_prompt, negative_prompt,
model_config_ref, conditioning_assets, generation_notes,
identity_motifs, secondary_motifs, materials, atmosphere,
color_direction, density, composition_notes, continuity_notes,
generation_margin_override_px, notes
```

`density` here is the *authored* density intent (e.g. "dense layered
tapestry"), distinct from the derived `context.block_grain` street-mesh
measure.

## 2. `extremity_regions.json` — 12 records

Same curated block as center records. Derived: polygons in panel-local and
global frames, bbox + gen window (global), optimizer facts (area fraction,
max angle), panel-local mask paths, same-panel neighbors, zigzag-seam
contact + associated seam ornament. `context.iconography` was **removed**
(v0.1 → v0.2): identity material is authored, so it lives in
`curated.identity_motifs` / `secondary_motifs` now. Legacy regional prompt
material will be imported into these curated fields via
`generation/references/extremity_prompt_legacy/` (nothing imported yet).

## 3. `seam_ornaments.json` — 2 records

`seam_left_laurel`, `seam_right_oak` — status `blocked_on_design`. Derived
geometry: zigzag polyline (global px), documented ink-overlap sizing facts.
Curated: the shared prompt-model fields (`concept_prompt`, `render_prompt`,
`negative_prompt`, `model_config_ref`, `conditioning_assets`,
`generation_notes`) plus design fields `branch_style`, `band_width_px`,
`opacity`, `generation_method` (all TBD), and `status`, `pilot`,
`style_pack`, `notes`.
